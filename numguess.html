    Number Guessing Game

    This project was was completed with the goal of implementing a MongoDB and routes for communication with the server. This application takes in a users name then creates a game record within the database and populates it with a random number. Then the user guesses a number from 1-100. The guess is relayed to the database where it is checked. The user is informed if they are too high or too low. If the user is correct the game ends and the high score table can be viewed. The high score table is filtered to only the top ten players with the shortest guesses, and have won the game. The biggest challenge with this application was the connection of the application to the MongoDB and correct implementation of routes.

    The Number Guessing Game focused on allowing connecting a users game with a MongoDB. In this MEN (MongoDB, Express.js, Node.js) application, a single clients plays a guessing game where the results are stored to a Mongo database.

    • MongoDB:
      for this application Node.js and Express were configured to handle HTTP requests and responses to interact with the server. The MongoDB was set up using a JSON game structure to store the game state and information.
    • Communication:
      Routes use HTTP methods (GET, POST, PUT, DELETE) that are called from the client side to access and perform different actions on the server. In this application route are handled as asynchronous functions that process incoming requests, perform logic, and generate responses. In order to achieve the desired out comes the routes include parameters that are dynamic values passed in from the client side in order to execute the game logic.
