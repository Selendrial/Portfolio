    Air BnB Clone

    This project was completed as a development exorcise using ASP.NET 8.0 and styled with Bootstrap 5.3.3 and msSQL. The entire project was developed by a team of 5 from scratch. this project focused on the development life cycle from concept to deployment. This project was a challenge to build a full stack application from the ground up. The project was chosen by the team and meetings began with the client. The client was consulted regularly for guidance and to understand user requirements. Once the requirements were established, development with asp.Net 8.0 was started. The AirBnB clone uses Identities and Entity core frameworks to scaffold the program as well as using a Microsoft SQL database backend. The system allows for booking of a property by a customer, and managing of the rental by a owning host.

    ASP.NET 8.0 is a web development framework developed by Microsoft that allows developers to build dynamic web applications. Additionally this project uses a Microsoft SQL database for or its infrastructure and data storage. This project was designed to be scalable and follow good object oriented programming methods.

    • ASP.NET Entity Framework:
      ASP.NET Entity Framework (EF) is an object-relational mapping (ORM) framework that allows for data access in ASP.NET applications. The EF is used to access the databases using object-oriented techniques, where database tables are represented as classes, and database operations are performed using LINQ queries.
    • ASP.NET Identity:
      ASP.NET Identity provides built-in functionality for user authentication, authorization, and account management in ASP.NET applications. The identity Framwoek integrates authentication and authorization functionalities, such as user registration, login, roles, password hashing, and account recovery.
    • MS SQL:
      The Microsoft SQL Server (msSQL) was used to support the ASP.Net Models as data tables allowing for the configureation of attribures and object-relational connections to be established for how the different tables interacted. The msSQL was used to store instance of a given object while allowing the objects to be queryable for dynamic display to the user.
