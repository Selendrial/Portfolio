    Speed Card Game

    This project was completed as a development exercise using web sockets for asynchronous communication. This project used a Mongo Express React Node (MERN) stack and implemented web sockets for seamless communication between the client and server. The biggest challenge of this program was the implementation of communication between the clients and server through sockets and dynamically drawing the cards as they are played.

    The Speed Card Game focused on allowing connections between multiple and a server to allow for the users to play games of speed. The game is played with a deck of 52 cards, 13 for each of the 4 suits (Hearts, Diamonds, Clubs, Spades). The game is played with 2 to 6 players and the goal is to be the first player to get rid of all the cards in their hand. The game is played by drawing cards from the deck and playing them in a stack face down. The player with the highest card in their hand wins the round and gets to draw 2 cards from the deck. The game is played until one player has won all the cards in the deck. Upon winning the users are asked for a player name which is then stored in a MongoDB database. The user is then shown the status of the played game.
